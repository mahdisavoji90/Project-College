# ساختار پروژه بازیافت نوین (Baziaft234)

این پروژه یک وب‌سایت خرید و فروش آنلاین ضایعات (الکترونیک، فلزات، پلاستیک، کاغذ) با تم تیره و قابلیت‌های کاربری و مدیریتی است.

---

## 📁 ساختار کلی دایرکتوری‌ها

```
baziaft234/
├── app/                    # هسته اصلی برنامه (MVC)
│   ├── Controllers/        # کنترلرها
│   ├── Core/               # کلاس‌های اصلی (مسیریاب، دیتابیس)
│   └── Models/             # مدل‌ها (خالی)
├── config/                 # تنظیمات برنامه
├── image/                  # تصاویر عمومی
├── public/                 # فایل‌های عمومی (خالی)
├── routes/                 # فایل‌های مسیریابی
├── uploads/                # فایل‌های آپلود شده
│   └── profiles/           # عکس‌های پروفایل کاربران
├── views/                  # ویوها (صفحات نمایشی)
│   ├── admin/              # صفحات پنل مدیریت
│   └── uploads/            # آپلودهای مرتبط با ویو
├── .env.example            # نمونه فایل تنظیمات محیطی
├── .htaccess               # تنظیمات Apache (URL Rewriting)
├── BACKEND_STRUCTURE.md    # مستندات ساختار بک‌اند
├── baziaft.sql             # فایل دامپ دیتابیس
├── bootstrap.php           # بوت‌استرپ برنامه
├── db.php                  # پل سازگاری backward
├── index.php               # فرانت کنترلر اصلی
└── router.php              # مسیریاب جایگزین
```

## 📂 توضیحات دایرکتوری‌ها و فایل‌ها

### 🌐 ریشه پروژه (Root)

| فایل | توضیحات |
|------|---------|
| `index.php` | **فرانت کنترلر اصلی**. تمام درخواست‌های HTTP از طریق `.htaccess` به این فایل هدایت می‌شوند. این فایل بوت‌استرپ را بارگذاری کرده، مسیریاب را ایجاد و مسیر درخواستی را پردازش می‌کند. |
| `router.php` | **مسیریاب جایگزین**. از طریق `APP_BASE_URL` مسیر پایه را تشخیص داده و درخواست را به ویو مناسب هدایت می‌کند. (برای استفاده مستقیم با آدرس‌های قدیمی) |
| `bootstrap.php` | **بوت‌استرپ برنامه**. session را آغاز می‌کند (`session_start`)، اتصال دیتابیس را با استفاده از PDO Singleton برقرار می‌کند (از طریق `App\Core\Database`)، ثابت `BASE_PATH` و `APP_BASE_URL` را تعریف می‌کند. همچنین توابع کمکی `url()`، `redirect()`، `require_user()` و `require_admin()` را تعریف می‌کند. |
| `db.php` | **پل backward-compatible**. برای سازگاری با کدهای قدیمی که مستقیماً `db.php` را فراخوانی می‌کردند، این فایل `bootstrap.php` را بارگذاری می‌کند. |
| `.htaccess` | تنظیمات Apache برای **URL Rewriting**. تمام درخواست‌هایی که به فایل یا دایرکتوری واقعی اشاره ندارند را به `index.php` هدایت می‌کند. نیازمند `mod_rewrite` و `AllowOverride All` است. |
| `.env.example` | نمونه فایل متغیرهای محیطی برای تنظیم دیتابیس (host, port, database name, username, password). |
| `baziaft.sql` | دامپ کامل دیتابیس شامل ساختار جداول و داده‌های اولیه. |
| `BACKEND_STRUCTURE.md` | مستندات تغییرات ساختار بک‌اند (قدیمی). |

### 📂 `app/` - هسته اصلی برنامه

#### `app/Controllers/`

| فایل | توضیحات |
|------|---------|
| `AdminController.php` | **کنترلر پنل مدیریت** (Namespace: `App\Controllers`). شامل متد `getStats($db)` که آمار کلی کاربران، درخواست‌های فروش و قیمت‌های بازیافت را از دیتابیس دریافت می‌کند. |

#### `app/Core/`

| فایل | توضیحات |
|------|---------|
| `Router.php` | **کلاس مسیریاب سبک**. یک مسیریاب ساده PHP که مسیرهای تعریف شده را با URI درخواستی مطابقت داده و فایل ویو مربوطه را بارگذاری می‌کند. از یک آرایه از مسیرها استفاده می‌کند و متد `any()` برای تعریف مسیر و `dispatch()` برای پردازش درخواست دارد. در صورت عدم تطابق، خطای 404 برمی‌گرداند. |
| `Database.php` | **کلاس اتصال دیتابیس** (Namespace: `App\Core`). از الگوی Singleton استفاده می‌کند. یک اتصال PDO با charset utf8mb4، حالت ارور Exception و حالت fetch Assoc ایجاد می‌کند. |

#### `app/Models/`

این دایرکتوری خالی است و هنوز مدلی در آن تعریف نشده است.

## 🔄 جریان کار (Flow)

1. کاربر آدرسی را در مرورگر وارد می‌کند.
2. `.htaccess` درخواست را به `index.php` هدایت می‌کند (URL Rewriting).
3. `index.php` فایل `bootstrap.php` را بارگذاری می‌کند:
   - Session شروع می‌شود.
   - اتصال دیتابیس برقرار می‌شود.
   - توابع کمکی تعریف می‌شوند.
4. یک شی `Router` ایجاد می‌شود.
5. فایل `routes/web.php` تمام مسیرها را در رجیستر می‌کند.
6. متد `dispatch()` مسیر درخواستی را با مسیرهای ثبت‌شده تطبیق داده و ویو مربوطه را از `views/` بارگذاری می‌کند.

  - `users` - اطلاعات کاربران

  - `recycling_prices` - قیمت‌های ضایعات
  - `transactions` - تراکنش‌های مالی
  - `addresses` - آدرس‌های کاربران
  - `notifications` - اعلان‌ها
  - و جداول دیگر...

## 🛠️ تکنولوژی‌ها و ابزارها

- **زبان:** PHP 8+ (با `declare(strict_types=1)`)
- **دیتابیس:** MySQL/MariaDB
- **اتصال دیتابیس:** PDO با الگوی Singleton
- **مسیریابی:** ر outer سفارشی و سبک PHP
- **Frontend:** HTML, CSS (تم تیره), JavaScript (Vanilla)
- **نمودار:** Chart.js (در پنل مدیریت)
- **آیکون‌ها:** Font Awesome 6
- **سرور:** Apache با mod_rewrite
- **مدیریت وابستگی:** ندارد (Pure PHP)

## 🔐 سشن و احراز هویت

- **کاربران:** از طریق `$_SESSION['user_id']` و `$_SESSION['user_email']` شناسایی می‌شوند.
- **ادمین:** از طریق `$_SESSION['admin_id']` و `$_SESSION['admin_username']` شناسایی می‌شوند.
- **توابع کمکی:** `require_user()` کاربر را به صفحه ورود هدایت می‌کند اگر لاگین نکرده باشد. `require_admin()` ادمین را به صفحه ورود مدیریت هدایت می‌کند.
- **رمز عبور:** با `password_hash()` و `password_verify()` (Bcrypt) ذخیره و بررسی می‌شود.
